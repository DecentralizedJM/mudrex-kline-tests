h2. Known issue — GET INR spot balance

{{GET /wallet/funds?trade_currency=INR}} returns {{200 OK}} with all fields {{0}} even when:

* {{89.12}} INR was successfully transferred SPOT → FUTURES
* {{14.67}} INR was successfully transferred FUTURES → SPOT (transfer {{Completed}}, futures balance decreased)

{{POST /futures/transfers/inr}} and {{GET /futures/funds?trade_currency=INR}} work correctly. Do not use GET INR spot for balance checks until fixed.

----

h2. API conventions

|| Request type || {{trade_currency}} placement ||
| GET | Query: {{?trade_currency=INR}} |
| POST / PATCH / DELETE | Body: {{"trade_currency": "INR"}} |
| Omitted | Defaults to {{USDT}} |

*Currency model:* Same USDT-quoted contracts ({{BTCUSDT}}, {{SOMIUSDT}}). Prices and liquidation in USDT. Margin, PnL, fees in INR. FX rate returned as {{hedge_rate}} / {{entry_hedge_rate}} / {{exit_hedge_rate}}.

h2. Implementation notes for developers

|| Topic || Detail ||
| INR transfers | Use {{POST /futures/transfers/inr}} — not {{/wallet/futures/transfer}} |
| INR spot balance | {{GET /wallet/funds?trade_currency=INR}} broken — do not use for balance |
| PATCH edit SL/TP | Include {{stoploss_order_id}} and {{takeprofit_order_id}} |
| PATCH amend order | Include both {{order_price}} and {{quantity}} |
| LIMIT orders | Notional must meet {{min_notional_value}} (5 USDT) at limit price |
| FX rate | {{hedge_rate}} on orders; {{entry_hedge_rate}} / {{exit_hedge_rate}} on positions |
| Prices | Order price, entry, liquidation — USDT. Margin, PnL, fees — INR |

----

h2. Conclusion

33 of 34 tested endpoints pass. The single failure is {{GET /wallet/funds?trade_currency=INR}}, which returns HTTP 200 but does not reflect the actual INR spot wallet balance. All other INR futures capabilities — transfers, trading, positions, orders, fees — work correctly on production.
