PATCH edit SL/TP — PASS (200)

Requires stoploss_order_id and takeprofit_order_id in the request body.

curl -X PATCH "https://trade.mudrex.com/fapi/v1/futures/positions/{position_id}/riskorder" \
  -H "Content-Type: application/json" \
  -H "X-Authentication: your-secret-key" \
  -d '{
    "stoploss_price": "0.095",
    "takeprofit_price": "0.14",
    "stoploss_order_id": "019ecc5d-d102-7c2e-a1ee-84df95bcc2c2",
    "takeprofit_order_id": "019ecc5d-d102-7c31-97cb-33d44488c69b",
    "is_stoploss": true,
    "is_takeprofit": true,
    "trigger_type": "MARKET"
  }'
{
    "success": true,
    "data": {
        "message": "Risk order amended successfully"
    }
}

SL updated 0.10 → 0.095, TP updated 0.13 → 0.14 (USDT prices).

PATCH amend order — PASS (200)

Requires both order_price and quantity in the request body (price-only amend fails).

# 1. Place LIMIT order
curl -X POST "https://trade.mudrex.com/fapi/v1/futures/SOMIUSDT/order?is_symbol" \
  -H "Content-Type: application/json" \
  -H "X-Authentication: your-secret-key" \
  -d '{
    "leverage": "5",
    "quantity": "44.0",
    "order_price": "0.11400",
    "order_type": "LONG",
    "trigger_type": "LIMIT",
    "trade_currency": "INR"
  }'

# 2. Amend order
curl -X PATCH "https://trade.mudrex.com/fapi/v1/futures/orders/{order_id}" \
  -H "Content-Type: application/json" \
  -H "X-Authentication: your-secret-key" \
  -d '{
    "order_price": "0.11380",
    "quantity": "44.0"
  }'
{
    "success": true,
    "data": {
        "message": "Order updated successfully"
    }
}

Price updated 0.114 → 0.1138 on verified GET.
